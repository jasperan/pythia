The Problem

Search engines got worse.
AI wrappers got expensive.

  • Perplexity Pro costs $20/month and sends your data to third-party APIs

The Solution

Pythia: your private oracle

A self-hosted Perplexity + Deep Research replacement combining SearXNG (free, unlimited web search), Ollama (local LLM inference), and Oracle AI Vector Search (semantic caching and knowledge accumulation).

Single-Shot Search

Cache-first, parallel-prefetch

1 Parallel prefetch fires cache lookup and SearXNG search concurrently
2 Cache HIT (similarity >= 0.85): cancel web search, return instantly
3 Cache MISS: web search is already running; synthesize via Ollama with citations
4 Store answer + embedding in Oracle (reuses the embedding from cache check)
5 Stream tokens via SSE in real-time with citation density in the DONE event

Deep Research

Autonomous multi-step research agent

Like Perplexity Deep Research or Google Deep Research, but fully self-hosted. The agent iteratively searches, analyzes gaps, and synthesizes comprehensive reports.

pythia research "What are the tradeoffs between RISC-V and ARM for edge AI?"

Research Pipeline

Seven steps to synthesis

1 Decompose: LLM breaks question into 3-5 focused sub-queries
2 Search & Scrape: each sub-query searched in parallel via SearXNG
3 Summarize: LLM extracts key findings from each sub-query's results
4 Analyze Gaps: LLM reviews all findings, identifies what's missing
5 Iterate: generates follow-up searches autonomously (up to N rounds)
6 Recall: Oracle vector search finds related findings from past sessions
7 Synthesize: produces a structured, cited research report

The Advantage

Your Pythia gets smarter over time

Every research session stores individual findings as embeddings in Oracle. When you research "RISC-V for edge AI" next month, Pythia recalls relevant fragments from your earlier research on "ARM vs x86 power efficiency".

No stateless search engine can do this.

Getting Started

5 steps to running Pythia

1 conda create -n pythia python=3.12 -y
2 pip install -e ".[dev]"
3 docker compose up -d (Oracle 26ai + SearXNG)
4 Run setup_schema.sql (optional ONNX loading only for experiments)
5 pythia search (launches TUI with auto-start)

Shortcut

Or just run one command

curl -fsSL https://raw.githubusercontent.com/
  jasperan/pythia/master/install.sh | bash

Clones the repo, creates a local virtualenv, and installs Python dependencies. Docker, Oracle schema setup, and Ollama still follow the README quick-start steps.

Streaming Protocol

Research SSE event stream

event: status   # "Planning research..."
event: plan     # sub_queries: ["q1","q2","q3"]
event: round_start # round: 1, max_rounds: 3
event: finding   # sub_query + summary_preview
event: gap_analysis # sufficient: false, gaps
event: recall    # count: 2, past findings
event: token     # streaming report content
event: done      # rounds_used, findings, elapsed

Why Oracle

Not just a database. An AI engine.

  • Vector storage + similarity search: Oracle stores embeddings and serves COSINE lookups efficiently
  • VECTOR column type with native COSINE distance (no pgvector extension drama)
  • SQL + Vector in one query: join cache lookups with history filters in a single statement
  • Oracle 26ai Free: runs in Docker, no license needed, full Vector Search included
